# AI 投资助手项目总览

## 文档头
- 角色：主文档
- 状态：当前有效
- 主文档：本页即主文档
- 用途：统一收口 AI 投资助手项目的项目说明、近期重构、测试方案、内容监控部署和 MVP 执行上下文

## 一句话结论
这个项目当前有 5 条主线：
1. **产品主线**：面向家人和中老年用户的 AI 投资助手
2. **技术主线**：Next.js + Mastra + tools + 会话持久化
3. **内容主线**：大 V 观点和财经内容进入站内结构化链路
4. **测试主线**：页面、路由、Tool、Agent 链路分层测试
5. **运维主线**：财经新闻监控系统部署在 `root81`，定时产出可复用 JSON

## 一、项目定位

### 1. 用户是谁
当前目标用户是家人和中老年用户。

### 2. 解决什么问题
核心问题有 4 个：
- 自选股和基金信息分散
- 行情、新闻、观点不容易整合
- 大模型问答容易悬浮，缺少真实数据支撑
- 普通用户看不懂复杂财经信息

### 3. 当前产品目标
当前产品目标：
- 让用户低门槛查看自选股、基金、市场概览
- 让用户直接用自然语言提问
- 让回答尽量基于真实行情、新闻和观点数据
- 让内容供给持续进入系统，不依赖手工临时复制

## 二、当前技术结构

### 1. 前端和应用层
- 框架：Next.js
- 语言：TypeScript
- UI：Ant Design + 自定义样式层
- 数据获取：SWR

### 2. Agent 层
- Agent 定义在 `src/mastra/agents/investment-agent.ts`
- Mastra 负责工具编排和模型调用
- 当前 Agent 主要围绕投资顾问场景组织回答

### 3. Tools 层
当前已经封装的能力包括：
- 股票 / 基金查询
- 大盘概览
- 风险评估
- 股票分析
- 新闻分析
- 热点分析
- 大 V 观点分析

关键点：
- 每个工具单独封装
- 工具输出结构稳定
- Agent 负责决定调用顺序和组织回答

### 4. Chat 链路
当前聊天链路：
`页面消息 -> /api/chat -> investmentAgent -> tools -> 回答写回会话`

这条链路已经具备：
- 登录校验
- 会话持久化
- thread / session 维持
- fallback 兜底

## 三、当前内容供给链路

### 1. 站内内容能力
系统已经支持：
- 大 V 文章导入
- 文章结构化分析
- 作者维度聚合展示
- Agent 直接消费这些内容

### 2. 财经新闻监控系统
内容监控系统部署在：
- 服务器：`root81`
- 路径：`/root/apps/finance-news-monitor`

当前支持的输入源：
- PDF
- 图片
- 手动导入文章文本
- 公众号文章
- 夸克网盘监控目录

### 3. 这条链路现在会产出什么
当前产出包括：
- 单篇分析 JSON / Markdown
- 按作者聚合的草稿
- AI 投资助手可直接复用的结构化 JSON
- 每日摘要输出

### 4. 当前联动方式
当前联动顺序：
- 外部内容进入 `finance-news-monitor`
- 结构化结果导出到 `exports/ai-investment-assistant`
- 站内脚本和接口再把这些内容导入 AI 投资助手

## 五、当前测试策略

### 1. 测试覆盖目标
项目当前的测试主线分成 5 层：
1. 纯逻辑函数
2. Mastra Tools 契约测试
3. API Route 契约测试
4. Hook / 组件 / 页面状态测试
5. Smoke 回归流程

### 2. 重点要测什么
当前优先守住这些链路：
- 行情接口变化后是否还能稳定返回
- Tool 输出结构是否稳定
- chat route 出错时是否能优雅降级
- watchlist、股票详情、基金详情是否有空态 / 错态处理
- 关键用户路径能不能完整走通

### 3. 当前推荐测试栈
- Vitest
- Testing Library
- MSW
- `vi.mock`

### 4. 当前建议命令
- `test`
- `test:coverage`
- `test:tools`
- `test:routes`
- `test:ui`
- `test:smoke`
